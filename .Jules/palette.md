## 2024-05-18 - Accessibility Improvements for HTML Report Visualizations
**Learning:** Static HTML visualizations like progress bar charts built with standard divs lack semantic meaning for screen readers. Using `role="progressbar"` along with `aria-valuenow`, `aria-valuemin`, and `aria-valuemax` adds missing context without requiring an actual `<progress>` element. Furthermore, icon-only representations (like decorative emojis for details such as file path or line number) must either be hidden (`aria-hidden="true"`) or explicitly labeled to provide meaningful context to assistive technologies.
**Action:** When creating custom UI components for data visualizations or status indicators using standard DOM elements, always include the relevant ARIA roles and value attributes. Hide purely decorative elements from screen readers while providing semantic text equivalents via `aria-label` or wrapping spans with tooltips.
